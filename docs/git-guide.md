# 🌊 VibeBoard 개발 및 Git 연동 가이드

브이와 함께하는 깔끔한 개발과 Git 관리 공식 가이드입니다.

## 1. 초기 세팅 (최초 1회)

새로운 프로젝트를 시작할 때 딱 한 번만 실행합니다.

```bash
# 1. Git 저장소 생성
git init

# 2. 모든 파일 추가 및 첫 저장
git add .
git commit -m "first commit"

# 3. 브랜치 이름 메인으로 설정
git branch -M main

# 4. GitHub 저장소 연결
git remote add origin https://github.com/사용자명/저장소명.git

# 5. GitHub로 업로드
git push -u origin main
```

## 2. 평상시 개발 워크플로우

개발을 진행하면서 변경사항을 저장할 때마다 반복하는 루틴입니다.

```bash
# 1. 모든 변경사항 스테이징
git add .

# 2. 커밋 메시지 작성 (feat: 신규 기능, fix: 버그 수정 등)
git commit -m "feat: 새로운 기능 추가"

# 3. GitHub에 업로드
git push
```

---

## 2. 자주 사용하는 Git 명령어

| 명령어                     | 용도                                   |
| :------------------------- | :------------------------------------- |
| `git status`               | 현재 프로젝트의 변경된 파일 상태 확인  |
| `git log --oneline`        | 지금까지의 커밋 내역 간단히 보기       |
| `git diff`                 | 현재 수정된 코드의 상세 변경 내용 확인 |
| `git checkout -b <branch>` | 새로운 기능 개발을 위한 브랜치 생성    |

---

## 3. ⚠️ 주의사항 (트러블슈팅)

### 중첩 Git 저장소 문제

프로젝트 폴더 내부에 다른 `.git` 폴더가 있으면 `git add .` 명령어가 제대로 동작하지 않고 `README.md`만 올리는 불상사가 생길 수 있습니다.

**해결 방법:**

1. 프로젝트 내부의 숨겨진 `.git` 폴더가 있는지 확인
2. 있다면 삭제 (`rm -rf 폴더/.git`)
3. `git add .` 명령어로 전체 파일 다시 추가

---

## 4. 커밋 메시지 컨벤션 (추천)

깔끔한 저장소를 위해 다음과 같은 접두어를 사용해 보세요.

- `feat:` 새로운 기능 추가
- `fix:` 버그 수정
- `docs:` 문서 수정 (README, docs 등)
- `style:` 코드 스타일 변경 (포맷팅 등)
- `refactor:` 코드 리팩토링
- `chore:` 빌드 업무 수정, 패키지 매니저 설정 등
